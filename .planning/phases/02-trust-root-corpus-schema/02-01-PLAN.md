Build the program's TRUST ROOT: an independently implemented, stdlib-only verifier that consumes a stored certificate record (plain JSON dict) and RAISES `VerificationError` on any structural or semantic violation — correct under `python -O`, sharing no logic with any searcher. Purpose: Everything downstream (schema, store, corpus, CI) rests on this one module. If it can be fooled, the whole epistemic contract collapses. It is proven adversarially in this plan, BEFORE any record is ever written (Plan 02's store calls it as the append gate).
Output: `src/alpha2/corpus/verifier.py` (trust root), `src/alpha2/invariants/witness.py` (emission-time extractor, untrusted), and four adversarial test suites.

is_conflict semantics (Appendix C.1, verbatim src/alpha2/search/heuristic.py and verify/model.py):
- is_conflict(A, B, adj) returns True iff branch sets A,B are NON-adjacent in G
  (every cross pair {x in A, y in B} is an H-edge, i.e. y in adj[x] for ALL pairs).
- Two branch sets are adjacent in G iff NOT is_conflict (some cross pair is a G-edge).

verify_model contract (Appendix E §4.6, hardened to raises): branch sets disjoint;
sizes in {1,2}; each size-2 set {a,b} is a G-edge (b NOT in adj[a]); all C(k,2)
cross-adjacencies present; k = len(sets) with k >= chi.

Canonical H_edges sha256 convention (frozen Phase 1 — tests/test_fingerprint.py::canonical_h_edges_sha256):
  edges = sorted([min(u,v), max(u,v)] for u<v); json.dumps(edges, separators=(",",":")); sha256 hexdigest.

|----------|-------------|

| emission toolkit (searcher/extractor) → verifier | The verifier shares NO logic with proposers; it re-derives conflict/adjacency/odd-components from stored H_edges and trusts nothing handed to it. |
