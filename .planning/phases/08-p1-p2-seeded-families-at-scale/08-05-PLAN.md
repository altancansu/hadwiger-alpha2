<objective>
Answer RESEARCH open-question A4 empirically: **measure** the ILP optimality-proof frontier for the
sum-free Cayley family instead of assuming it sits at n≈150–200. A small driver generates structured
gate-surviving instances across a group-order grid (n = 31, 61, 101, 151, 201, …), runs the exact
had₂ (and, on had₂<χ, the had₃) backends under a FIXED deterministic budget, and records per n
whether optimality PROVED. The result is the empirical boundary between the exact `g>0` window and
the RESISTANT-→-E3 region — a required input to the grid sweep (08-06) so it never mislabels a
heuristic miss as a `g>0` point.

Because "PROVED" here means BOTH co-equal backends reach PROVED_OPTIMAL, both are bounded
deterministically — CP-SAT by `det_time`, CBC by `det_nodes` (the 08-02 node-cap) — so the measured
frontier is a function of (n, budget), never of wall-clock contention on the shared box.

Purpose: make the sweep's exact-window edge data-driven (A4), and quantify the cost/rigor split the
whole phase is designed around.
Output: `pool/sumfree/frontier.py` + a stored frontier table (`data/results/`), test_frontier GREEN.
</objective>
